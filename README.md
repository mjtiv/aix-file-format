## üß† What Is a `.aix` File?

`.aix` stands for **AI eXecutable** ‚Äî a novel, plain-text file format designed to **encapsulate prompt logic, data, scoring rules, and executable scripts** into a single, portable, and LLM-friendly document.

It is:

- Human-readable ‚úÖ

## üß± Anatomy of an `.aix` File

Each file is made of **modular section headers** that divide content:

```
=== AIX:Meta ===         # Author, version, intent  
=== AIX:Rules ===        # Scoring rubrics, prompt conventions  
=== AIX:Founders_Prepopulated ===  # Structured JSON data (e.g., tone profiles)  
=== AIX:User_Interface ===          # AI-guided prompt flow  
=== AIX:Python_Script ===           # Executable code block (auto-runs on open)  
=== AIX:End ===          # Logical EOF marker  
```

> üìù **Note**: Not all sections are required in every `.aix` file.  
> This structure is **flexible and composable** ‚Äî a file might include only data and script, or just rules and UI flow. Think of it like a modular AI workbook: use only the blocks you need.
